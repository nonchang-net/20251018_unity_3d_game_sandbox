# 作業ログ - 20251101_2115

## 変更した内容の概要

GameCameraManagerのイベント購読をStart()からInitialize()メソッドに移動しました。これにより、コンパイル直後の初回再生時にZキーを押しても、トラッキング設定が準備される前にイベントが発火しないようになりました。

**修正ファイル:**
- `Assets/Scripts/Camera/GameCameraManager.cs` - Start()メソッドを削除、Initialize()にイベント購読を追加

## なぜそのように変更しようと考えたか

**問題の症状:**
前回の修正（20251101_2110）で初期化タイミング問題は改善しましたが、コンパイル直後の初回再生時にZキーを押すと以下のメッセージが出る問題が残っていました：
```
GameCameraManager: トラッキング設定が設定されていないため、カメラ切り替えできません。
```
2回目以降の再生では正常に動作していました。

**原因:**
実行順序が以下のようになっており、イベント購読とトラッキング設定の準備の間に隙間がありました：

1. `Start()` - `SubscribeCameraViewChangeEvents()` でイベント購読開始
2. `Start()` - `InitializeManagers()` コルーチン開始
3. **← この間にZキーを押すとエラー発生**
4. 次フレーム - `Initialize()` でトラッキング設定を適用

つまり、トラッキング設定が準備される前にイベント購読が開始されていたため、初期化完了前にZキーを押すことができてしまっていました。

**修正内容:**

```csharp
// 修正前
void Start()
{
    SubscribeCameraViewChangeEvents();
    SubscribeCameraLockAreaEvents();
}

public void Initialize()
{
    // 参照確認とトラッキング設定の適用
}

// 修正後
public void Initialize()
{
    // 参照確認とトラッキング設定の適用

    // イベント購読（トラッキング設定の準備が完了した後に開始）
    SubscribeCameraViewChangeEvents();
    SubscribeCameraLockAreaEvents();
}

// Start()メソッドは削除
```

**効果:**
- トラッキング設定が確実に準備された後にイベント購読が開始される
- コンパイル直後の初回再生時でも、Zキーが正常に動作する
- 初期化処理が1つのメソッドに集約され、処理の流れが明確になる
